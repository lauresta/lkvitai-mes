# Incident Response Runbook

## Procedure 1: Severity Classification
1. Classify incident as Sev1/Sev2/Sev3 based on user impact and availability.
2. Assign incident commander and communications owner.
3. Start incident timeline with UTC timestamps.
4. Initiate bridge channel and on-call paging.

## Procedure 2: Escalation Workflow
1. Page L1 immediately for Sev1/Sev2 alerts.
2. Escalate to L2 if unresolved in 15 minutes.
3. Escalate to L3/management if unresolved in 30 minutes.
4. Document each escalation action in timeline.

## Procedure 3: Stakeholder Communication
1. Publish initial status update within 15 minutes.
2. Provide impact scope, workaround, and ETA.
3. Continue updates every 30 minutes while active.
4. Publish final resolution summary after recovery.

## Procedure 4: Post-Incident Review
1. Create postmortem within 24 hours.
2. Document root cause, contributing factors, and recovery steps.
3. Track follow-up actions with owners and due dates.
4. Link postmortem to runbook updates where needed.
