Designer Handoff: MudBlazor Migration

This is the single static file to hand to the designer. The designer may edit this file directly or return annotated recommendations. Codex will later distribute the final decisions into AppTheme.cs, site.css, shared components, and migration rules.

Assignment For Designer
You are reviewing the LKvitai.MES Warehouse WebUI MudBlazor migration.

Goal:
Align the current MudBlazor bridge layer and future MudBlazor migration rules with the already approved LKvitai.MES design system. Do not invent a new visual language.

Primary design source:
- docs/ux/lkvitai-mes-ux-handoff.html
- portal-tokens.css
- portal-shell.css

Approved visual baseline:
- compact enterprise/product UI
- dark Portal topbar/header
- LKvitai.MES brand block
- neutral grey application surfaces
- teal active/focus/link/primary states
- warm sand only for operational warning/scaffold accents
- dense operational tables
- no marketing layout
- no decorative gradients
- no random bright colors
- no oversized cards

Please edit this handoff or return a written answer with these sections:

A. AppTheme.cs recommendations
   - final MudBlazor palette mapping
   - radius/elevation defaults
   - any MudBlazor theme properties that should be added

B. site.css additions/replacements
   - panels / MudPaper
   - dialogs
   - error, warning, info, success states
   - buttons
   - forms and focus states
   - chips and statuses
   - dense tables / MudDataGrid
   - pagination, toolbars, filters
   - empty/loading states

C. MudBlazor component usage rules for future pages
   - which Mud components to use
   - which Mud defaults must be overridden
   - component density, spacing, and table rules

D. Bootstrap migration rules
   - how to replace Bootstrap cards, rows, buttons, forms, tables, alerts, badges
   - how to replace Bootstrap Icons
   - what must stay during Phase 1-2 coexistence

E. Current conflicts
   - CSS or markup that conflicts with the approved design system
   - risk areas before Bootstrap is removed in Phase 3

Important:
Do not redesign the app. Align it to the existing approved design. Keep it compact, operational, dense, and enterprise-focused.
