🎨 Design questions for Claude

warehouse shell
Q1 · LAYOUT
Does the left sidebar belong inside or outside the shared topbar container?
Portal/Sales/Frontline have no persistent left nav — the shared PortalModuleShell just wraps full-width content. Warehouse is different: it has a 250 px collapsible sidebar (MudBlazor Drawer) for 8 nav sections with 50+ items. Should the sidebar sit below the topbar as a column in a flex row, or should we follow a different pattern entirely for WMS?
Q2 · NAV ACTIVE COLOUR
Yellow or teal for active nav items?
Current Warehouse nav uses #facc15 (yellow) for active state — this preview switches it to --accent-300 (teal). Teal is the design system primary. Yellow isn't in the token set at all. Does yellow stay (for differentiation from Portal/Sales), or standardise on teal?
Q3 · NAV HAMBURGER
Where does the sidebar toggle button live in the new topbar?
The current topbar has a Bootstrap hamburger button that toggles the MudDrawer. The shared topbar has no slot for module-level controls. Three options: (a) add a left slot to PortalModuleShell for a toggle icon, (b) keep hamburger only on mobile (sidebar is always visible on desktop ≥ 1024 px), (c) replace MudDrawer with a CSS-only sidebar that doesn't need a toggle.
Q4 · INFORMATION ARCHITECTURE
8 sections, 50+ items — is that too many for a left nav?
The warehouse has: Stock · Inbound · Outbound · Operations · Finance · Admin · Reports · Analytics. Admin alone has 22 items. On a 768 px screen this nav is taller than the viewport. Should some sections (Admin, Reports) move to a separate settings/reporting area instead of the primary left nav?
Q5 · SIDEBAR WIDTH & FEEL
250 px sidebar width — keep or adjust?
Current MudDrawer is 250 px. The content area for dense operational tables (stock balance, picking tasks) benefits from being as wide as possible. Would 200 px or icon-only collapsed state (48 px) + hover-expand work better for power users who know the nav? Or is full label always visible preferred?
Q6 · BOOTSTRAP vs TOKEN SYSTEM
Bootstrap CSS — remove entirely or just override?
Warehouse ships Bootstrap 5 + MudBlazor. Portal/Sales use only MudBlazor + token CSS. Bootstrap adds ~200 KB and resets conflict with MudBlazor. For the new shell: strip Bootstrap and use the token system (already done in this preview for the nav), or keep Bootstrap as a utility layer for the content pages (tables, forms, badges)?
